Run Stage 1 (Empathize) from the Design Thinking skill at `~/.claude/skills/design-thinking/SKILL.md`.

1. Identify the primary user types for the feature/system being discussed
2. Build an empathy map for each user type (THINKS, FEELS, SAYS, DOES, PAIN, GAIN)
3. Consider contextual factors (technical literacy, language/culture, accessibility, usage frequency)
4. Deliver a pain point priority list ranked by severity and frequency

Read the project's master doc and relevant code first. Ask the user what feature or area to focus on if not specified.
