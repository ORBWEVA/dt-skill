---
name: dt:audit
description: Audit an existing feature or page against design thinking principles (scorecard)
---

Run Audit Mode from the Design Thinking skill at `~/.claude/skills/design-thinking/SKILL.md`.

Review the specified feature, page, or system against 5 criteria:

1. **User Clarity** (1-5) — Is it clear who this is for?
2. **Problem Definition** (1-5) — Is the problem clearly stated?
3. **Solution Fit** (1-5) — Does the solution match the problem?
4. **Validation** (1-5) — Has this been tested with users?
5. **Iteration** (1-5) — Has it been refined based on feedback?

Read the relevant code and docs first. Output the DT AUDIT scorecard with score out of 25 and top recommendation.
