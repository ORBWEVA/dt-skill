# Design Thinking Commands

| Command | Stage | What it does |
|---------|-------|-------------|
| `/dt full` | All | Walk through all 5 stages for a feature, system, or product |
| `/dt empathize` | 1 | User research: personas, empathy maps, pain point mapping |
| `/dt define` | 2 | Synthesize a HMW problem statement + success criteria |
| `/dt ideate` | 3 | Brainstorm with constraint-breaking + prioritization matrix |
| `/dt prototype` | 4 | Generate specs, wireframes, or code prototypes |
| `/dt test` | 5 | Create test plans, feedback frameworks, decision criteria |
| `/dt audit` | Review | Score an existing feature against DT principles (1-25) |
| `/dt help` | — | This help screen |

## Workflow Integration

- Before `/gsd:create-roadmap` → `/dt define` to sharpen the problem
- Before a new feature → `/dt empathize` to clarify who it's for
- After shipping → `/dt audit` to find iteration opportunities
- Full exercise → `/dt full` for comprehensive design thinking

## Principles

1. Users first, technology second
2. Diverge before converging
3. Make it tangible (prototype early)
4. Test assumptions, not egos
5. Iterate, don't perfect
